LeetCode142 环形链表2

环形链表2，需要用到同余运算（Modular Arithmetic）的知识，本题涉及到两个证明：

  1. 如何证明fast和slow指针一定会相遇
  2. 如何证明使用ptr指针再走a个距离正好可以和slow指针在链表进入处相遇

Modular Arithmetic

$A \equiv B(\bmod C)$ 意味着 A 和 B 在 mod C 的系统里是相等的，即 A 和 B 除 C 有相同的余数。举个例子，$3 \equiv 5 (\bmod 2)$ 意味着在 mod 2 系统中 3 是等于 5 的，因为 $3%2=1, 5%2=1$ 这两者的余数都为 1，所以他们在模 2 系统中是等价的。注意这里的 $\equiv$ 与 $=$ 并不等价，$5 \equiv 3(\bmod 2)$ 读作 5 congruent to 3 given module 2。当 $A \equiv B(\bmod C)$ 时，有以下几个推论：

  1. A 和 B 有同样的余数，当他们同时除以 C 时
  2. $A = K\times C +B, K\in \mathbb Z$
  3. $C|(A-B)$ A 与 B 的差可以被 C 整除，或者说 A 与 B 的差是 C 的整数倍

证明 fast 与 slow 指针一定会相遇

设链表环外的长度为 a，环长为 n，slow 指针的速度为 1，fast 指针的速度为 2，则经过 $t$ 时间，slow 指针走过的长度为 $t$，fast 指针走过的长度为 $2t$，slow 指针在环上所处的位置为 $(t-a)%n$，fast 指针在环上所处的位置为 $(2t-a)%n$，如果这样的 t 存在，则两指针一定可以相遇，t 要求是整数（非整数一定可以相遇，则该解是 trivial 的），以下是推导过程： $$ \begin{gather} t-a \equiv 2t-a (\bmod n) \\\
n | t (\bmod n) \\\
t = k\cdot n \qquad k\in\mathbb Z \end{gather} $$ 所以此时只要取 t 为 n 的整数倍时两指针就能相遇。该问题可以推广到 fast 指针的速度为 m 的情况，此时 $n|(m-1)t$ 即 $t = \frac{kn}{m-1}$，此时取 k 为 m-1 时即可以保证 t 是整数，所以对于任意 m 上述结论同样成立。在 leetcode 上验证 fast 速度为 3 时同样成立。

class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        if head is None or head.next is None:
            return False
        fast = head.next.next
        slow = head.next

        while slow != fast:
            if fast is None or fast.next is None:
                return False
            fast = fast.next.next
            slow = slow.next
        
        return True

注意代码中的 trick，当两个指针相遇时就肯定有环，当 fast 指针可以走到终点时就肯定没有环。

这里还有一个需要注意的地方，就是如果没有这段环外路径，两个点在环上任意位置处开始，则这两个点是不一定相遇的，证明如下：

假设 fast 指针在 $x$ 位置处，slow 指针在 $0$ 位置处，则经过 $t$ 时间，fast 指针所在的位置是 $x + mt$，slow 指针的位置在 $t$，此时要判断是否存在 $t$ 满足： $$ \begin{gather} x+mt \equiv t ~~(\bmod n) \\\
n|x+(m-1)t ~~(\bmod n) \\\
x+(m-1)t = k\cdot n \\\
t = \frac{kn-x}{m-1} \end{gather} $$ 此时 t 不一定对于所有的 m 和 x 都有解。为什么这种情况与上述问题不一样呢，原因是在上述问题中，当 slow 指针走过 a 的距离到达链表入口时，fast 指针多走的距离是 $(m-1)a$，用这个值替换上面式子中的 $x$，可以得到: $$ t=\frac{kn-(m-1)a}{m-1}=\frac{kn}{m-1}-a $$ 此时只要取 $k$ 为 $m-1$ 的倍数时，则 $t$ 一定存在整数解。

计算 fast 和 slow 指针的相遇点

经过 t 时间，fast 指针和 slow 指针在环上所在位置相等，这是上一个证明所用的等式。另一个等式是 fast 指针和 slow 指针所走过的路程是两倍关系。这里用的是第二个等式。

其中 slow 指针走过的长度为 $a+b+\alpha(b+c)$，fast 指针走过的长度为 $a+b+\beta(b+a)$，因为 fast 指针走过的长度是 slow 指针的两倍，所以有以下关系式： $$ \begin{gather} a+b+\beta(b+c) = 2(a+b+\alpha(b+c)) \\\
a+b+\beta(b+c) = 2a+2b+2\alpha(b+c) \\\
0 = a+b+(2\alpha-\beta)(b+c) \\\
a=(\beta-2\alpha)(b+c)-b\\\
a=(\beta-2\alpha-1)(b+c)+c \end{gather} $$ 此时，$a$ 的长度是 $(b+c)$ 的倍数再加上 $c$，而 c 正好是 slow 指针离链表入口的距离，slow 指针再走 c 步正好到链表入口，再走 $(\beta-2\alpha-1)(b+c)$ 步还是会在链表入口。所以我们可以定义一个指向链表头部到指针 ptr，让 ptr 和 slow 同时以1为步长前进，这样当这两个指针相遇的时候，slow 指针还会在链表入口处。

class Solution:
    def detectCycle(self, head: ListNode) -> ListNode:
        if head is None or head.next is None:
            return None
        
        fast = head.next.next
        slow = head.next
        ptr = head

        while fast != slow:
            if fast is None or fast.next is None:
                return None
            fast = fast.next.next
            slow = slow.next
        
        while ptr != slow:
            ptr = ptr.next
            slow = slow.next
        
        return ptr
